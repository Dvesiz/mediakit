## 1. 项目概述

### 1.1 项目定位

MediaKit 是一个**纯前端媒体工具箱**，提供图片/视频/音频处理能力，所有操作在用户的浏览器本地完成，不上传任何数据到服务器。

### 1.2 核心原则

| 原则 | 说明 |
|------|------|
| **零上传** | 所有数据在浏览器内处理，无后端服务器 |
| **隐私安全** | 文件不离开用户设备 |
| **离线可用** | PWA 支持，核心功能可离线运行 |
| **响应式** | PC / 平板 / 手机均良好体验 |
| **轻量快速** | 首屏加载控制在 3s 内 |

### 1.3 工具矩阵

| 编号 | 工具 | 分类 | 难度 | 优先级 | 依赖 |
|------|------|------|------|--------|------|
| 1 | 图片压缩 | 图片 | ⭐⭐ | P0 | 无（Canvas API） |
| 2 | 格式转换 | 图片/视频/音频 | ⭐⭐⭐ | P0 | Canvas / FFmpeg.wasm |
| 3 | 图片编辑 | 图片 | ⭐⭐ | P0 | Canvas API |
| 4 | 视频工具 | 视频 | ⭐⭐⭐⭐ | P1 | FFmpeg.wasm |
| 5 | 音频工具 | 音频 | ⭐⭐⭐ | P1 | FFmpeg.wasm / Web Audio API |
| 6 | 二维码工坊 | 二维码 | ⭐⭐ | P0 | qrcode + jsQR（已有代码） |
| 7 | 颜色实验室 | 色彩 | ⭐ | P1 | Canvas API |

> **P0** = 第一阶段必须完成 | **P1** = 第二阶段完成

## 2. 技术栈

### 2.1 核心依赖

| 技术 | 用途 | 版本 |
|------|------|------|
| **Vite** | 构建工具 | ^8.0 |
| **Vanilla JS** | 框架（无框架，轻量） | ES2022+ |
| **CSS Custom Properties** | 主题系统 | — |
| **Canvas 2D API** | 图片处理 | 浏览器原生 |

### 2.2 第三方依赖

| 包名 | 用途 | 预计体积（gzip） |
|------|------|-----------------|
| `qrcode` | 二维码生成 | ~15KB |
| `jsQR` | 二维码解码 | ~25KB |
| `@ffmpeg/ffmpeg` | 视频/音频处理 | ~850KB（WASM） |
| `@ffmpeg/util` | FFmpeg 工具函数 | ~5KB |

### 2.3 开发依赖

| 包名 | 用途 |
|------|------|
| `vitest` | 单元测试 |
| `gh-pages` | GitHub Pages 手动部署 |

### 2.4 架构决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 框架选型 | 无框架（Vanilla JS） | 避免框架体积，各工具独立，适合工具站场景 |
| 路由方案 | Hash 路由 + lazy import | 简单可靠，支持 SEO 不重要（工具站） |
| 样式方案 | CSS Variables + 全局 CSS | 无运行时开销，CSS 变量天然支持主题 |
| 状态管理 | DOM 驱动 + 模块局部状态 | 每个工具独立，无跨工具状态共享需求 |
| 视频/音频引擎 | FFmpeg.wasm | 唯一可在浏览器端运行的完整音视频编码方案 |
| 构建配置 | Vite 默认配置 | 零配置即可，按需调整 base 路径 |
| 部署 | GitHub Actions CI/CD | 推送即部署，无需手动操作 |


## 4. 工具规格

### 4.1 图片压缩工具 `image-compress`

**目标**：浏览器端无损/有损压缩图片，实时对比预览。

**功能列表**：

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 拖拽/点击上传 | 支持 PNG / JPG / WebP | P0 |
| 质量调节 | 滑块 1–100 控制压缩质量 | P0 |
| 实时预览 | 原图 vs 压缩图左右对比 | P0 |
| 显示信息 | 原始大小、压缩后大小、压缩率 | P0 |
| 一键下载 | 下载压缩后的图片 | P0 |
| 批量处理 | 一次选多张，逐张压缩 | P1 |
| 尺寸缩放 | 可选限制最大宽高 | P1 |

**技术实现**：
```
Canvas API:
  1. 读取文件 → FileReader / createImageBitmap
  2. 绘制到 Canvas
  3. canvas.toBlob(blob, type, quality) 输出
  4. URL.createObjectURL 预览
```

**性能目标**：
- 单张 < 5MB 图片压缩延迟 < 500ms
- 预览切换流畅无卡顿

**边界情况**：
- 极小图片（< 100px）：不压缩直接返回
- 超大图片（> 4000px）：自动等比缩放到 2000px 以改善性能
- 透明 PNG：转换为 PNG 格式避免丢失透明度
- 已高度压缩的图片：检测到压缩率 < 5% 时提示用户

---

### 4.2 格式转换工具 `format-convert`

**目标**：图片/视频/音频格式互转。

**功能列表**：

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 图片格式互转 | PNG ↔ JPG ↔ WebP ↔ BMP | P0 |
| 视频格式转换 | MP4 ↔ WebM ↔ AVI | P1 |
| 音频格式转换 | MP3 ↔ WAV ↔ OGG ↔ AAC | P1 |
| 批量转换 | 多文件统一转换 | P1 |
| 保持原尺寸 | 图片转换不改变尺寸 | P0 |
| GIF 输出 | 视频转 GIF | P1 |

**技术实现**：
```
图片：Canvas toBlob(type)
视频/音频：FFmpeg.wasm
  - 加载 FFmpeg core 到 Web Worker
  - 写入输入文件到虚拟文件系统
  - 执行 ffmpeg -i input output 命令
  - 读取输出文件
```

**限制说明**：
- 视频转换受限于浏览器 Codec 支持，H.265/HEVC 可能不可用
- 大视频文件（>500MB）可能因浏览器内存限制失败
- 音频转换通常比视频快，可放宽到 100MB

### 4.3 图片编辑工具 `image-edit`

**目标**：基础图片编辑功能。

**功能列表**：

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 裁剪 | 自由/固定比例裁剪（1:1, 4:3, 16:9） | P0 |
| 旋转 | 90°/180°/270° 旋转 | P0 |
| 翻转 | 水平/垂直翻转 | P0 |
| 缩放 | 按百分比缩放 | P0 |
| 基础滤镜 | 灰度 / 模糊 / 亮度 / 对比度 / 饱和度 | P0 |
| 撤销/重做 | 操作历史栈 | P1 |
| 自由绘图 | 画笔标注 | P2 |

**技术实现**：
```
Canvas 2D API + CSS filter 混合方案：
  - 变换操作（旋转/翻转/缩放）→ Canvas 变换矩阵
  - 像素滤镜（灰度/亮度/对比度/饱和度）→ getImageData + putImageData
  - 模糊滤镜 → canvas filter 属性
```

**操作历史**：
```
操作栈实现：
  - 每次操作前保存 Canvas 快照（toDataURL）
  - 撤销：pop 上一个快照并恢复
  - 重做：push 到 redo 栈
  - 限制历史栈最大 20 步，防止内存溢出
```

### 4.4 视频工具 `video-tools`

**目标**：视频压缩与格式转换。

**功能列表**：

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 视频压缩 | 按码率/分辨率/CRF 压缩 | P0 |
| 视频转 GIF | 选取片段转 GIF，可调帧率/尺寸 | P0 |
| 格式转换 | MP4 ↔ WebM ↔ AVI | P1 |
| 视频信息 | 显示分辨率/时长/编码/大小 | P0 |
| 进度显示 | 实时处理进度条 | P0 |

**技术实现（FFmpeg.wasm）**：

```
初始化 FFmpeg：
  1. import { FFmpeg } from '@ffmpeg/ffmpeg'
  2. import { fetchFile, toBlobURL } from '@ffmpeg/util'
  3. const ffmpeg = new FFmpeg()
  4. await ffmpeg.load({
       coreURL: await toBlobURL('/ffmpeg-core.js', 'text/javascript'),
       wasmURL: await toBlobURL('/ffmpeg-core.wasm', 'application/wasm'),
     })

视频压缩命令示例：
  ffmpeg -i input.mp4 -vcodec libx264 -crf 23 -preset fast output.mp4

视频转 GIF 命令示例：
  ffmpeg -i input.mp4 -vf "fps=10,scale=480:-1" -loop 0 output.gif
```

**限制与容错**：
- 文件 < 200MB 体验最佳
- 处理期间页面可能卡顿 → 使用 Web Worker 隔离（P1）
- 不支持加密/DRM 保护的内容
- 首次加载需下载 ~3MB WASM 文件

### 4.7 颜色实验室 `color-lab`

**目标**：从图片提取颜色、生成调色板、颜色格式互转。

**功能列表**：

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 图片取色 | 上传图片提取主色调（Top 5 颜色） | P0 |
| 调色板生成 | 提取 5–10 色和谐调色板 | P0 |
| 颜色格式互转 | HEX ↔ RGB ↔ HSL 实时互转 | P0 |
| 颜色预览 | 色块展示 + 复制到剪贴板 | P0 |
| 渐变色生成 | 双色/三色渐变 CSS 代码生成 | P1 |

**技术实现**：
```
颜色提取算法：
  1. 图片绘制到 Canvas（缩放到 200px 宽以提高性能）
  2. getImageData 获取像素数据
  3. 中值切割算法（Median Cut）提取主色调
  4. 按像素数量降序排列

颜色格式转换：
  HEX ↔ RGB：parseInt + toString(16)
  RGB ↔ HSL：标准颜色空间转换公式
```

### 5.2 数据流

```

                                  → 用户下载（无网络请求）
```

**关键点**：整个流程没有网络请求，所有数据在浏览器内存中流转。

### 5.3 FFmpeg.wasm 共享策略

视频和音频工具共享同一个 FFmpeg 实例：

```js
// src/common/ffmpeg.js（未来可能添加）
let ffmpegInstance = null

export async function getFFmpeg() {
  if (!ffmpegInstance) {
    const { FFmpeg } = await import('@ffmpeg/ffmpeg')
    const { toBlobURL } = await import('@ffmpeg/util')
    ffmpegInstance = new FFmpeg()
    await ffmpegInstance.load({
      coreURL: await toBlobURL('/ffmpeg-core.js', 'text/javascript'),
      wasmURL: await toBlobURL('/ffmpeg-core.wasm', 'application/wasm'),
    })
  }
  return ffmpegInstance
}
```

**加载时机**：仅在用户首次访问视频/音频工具时加载，首页不预加载。

## 8. 开发规范

### 8.1 JavaScript 规范

- **语言版本**：ES2022+（Vite 会自动转译）
- **模块格式**：ES Module（`import` / `export`）
- **命名规范**：
  - 变量/函数：`camelCase`
  - 常量：`UPPER_SNAKE_CASE`
  - 类/构造函数：`PascalCase`
  - 文件名：`kebab-case`
- **异步**：优先使用 `async/await`，避免裸 `.then()`
- **副作用**：函数越纯越好，DOM 操作集中在 `render` 函数内

### 8.2 CSS 规范

- **命名方式**：语义化 class 名，使用 `-` 连接（如 `.tool-card`、`.drop-zone`）
- **样式隔离**：工具专用样式在工具自己的 `style.css` 中，使用工具前缀避免冲突
- **主题**：使用 `var(--color-*)` CSS 变量，不要硬编码颜色值
- **响应式**：优先使用 CSS Grid / Flexbox，断点参考：
  - 手机：< 768px
  - 平板：768px–1024px
  - 桌面：> 1024px

### 8.3 Git 规范

- **分支策略**：`main` 为发布分支，功能开发在 feat/ 分支
- **提交信息**：简洁中文，格式 `[模块] 改动说明`
  ```
  [压缩] 添加批量处理功能
  [路由] 修复 hash 导航时首页不刷新
  [样式] 暗色模式下的边框颜色调整
  ```
- **提交粒度**：一个功能点一次提交，不要混合无关改动

### 8.4 文件规范

- 每个文件不超过 300 行，超过考虑拆分
- 每个函数不超过 50 行，超过考虑拆分
- `index.js` 只做导出和、render 函数，逻辑放到子模块

### 8.5 错误处理

- 所有用户操作必须有 try-catch，避免白屏
- 错误信息用中文展示给用户
- 关键操作（文件读写、FFmpeg 调用）需要详细错误日志

## 13. 常见问题

### Q: 为什么不用 React/Vue？
工具站各工具独立，无复杂状态共享，Vanilla JS 体积更小、加载更快，且与已有的二维码工坊技术栈一致。

### Q: 视频处理这么慢怎么办？
FFmpeg.wasm 在浏览器中的性能约为原生 1/3 到 1/5。建议：
- 文件限制在 200MB 以下
- 处理时显示清晰的进度条
- 未来可考虑 Web Worker 避免阻塞 UI

### Q: 隐私真的安全吗？
是的。所有代码在浏览器中执行，文件通过 `FileReader` 和 `URL.createObjectURL` 读取，不经过任何网络请求。FFmpeg.wasm 也是客户端 WASM 运行时。

### Q: 支持哪些浏览器？
支持所有现代浏览器（Chrome 90+ / Firefox 90+ / Safari 15+ / Edge 90+），不支持 IE。

### Q: 部署后访问地址是什么？
```
https://<你的 GitHub 用户名>.github.io/mediakit/
```
